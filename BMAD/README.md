# BMAD Documentation

**BMAD** (Brownfield Method for AI-Assisted Development) documentation provides comprehensive technical documentation
for the Directus codebase.

## What These Docs Are

These documents analyze the Directus system as a brownfield project, providing architecture insights, API references,
workflows, dependencies, and technical debt analysis to help developers understand and work with the codebase
effectively.

## Suggested Reading Order

1. **[SYSTEM_OVERVIEW.md](SYSTEM_OVERVIEW.md)** - Start here for high-level understanding
2. **[ARCHITECTURE.md](ARCHITECTURE.md)** - System architecture with diagrams
3. **[SETUP_INSTALLATION.md](SETUP_INSTALLATION.md)** - Getting started locally
4. **[CODE_STRUCTURE.md](CODE_STRUCTURE.md)** - Navigate the codebase
5. **[API_DOCS.md](API_DOCS.md)** - Complete API reference
6. **[USAGE_WORKFLOWS.md](USAGE_WORKFLOWS.md)** - Common workflows and examples
7. **[DEPENDENCIES.md](DEPENDENCIES.md)** - Libraries and external services
8. **[KNOWN_ISSUES_TECH_DEBT.md](KNOWN_ISSUES_TECH_DEBT.md)** - Critical issues and risks

Each document is self-contained but references others where relevant.
